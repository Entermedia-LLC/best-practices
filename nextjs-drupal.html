    Next.js for Drupal

    The purpose of this document is to help guide you through working with a potential project using the framework, Next.js for Drupal. Outside of the official documentation, you should find here personalized recommendations for using the framework based on the types of projects we typically see at Entermedia. We have broken these recommendations out into common elements we tend to interact with often in client work. If something you’re looking for isn’t represented you can either submit a pull request to this repository or refer to the official Next.js for Drupal documentation.

    Getting Started

    To begin, follow the instructions for creating a Next.js site in Next.js for Drupal's docs.

    Install commitlint

    For projects using Git, install commitlint.

    Install Sass

    Entermedia uses Sass to compile CSS. Next.js allows you to import Sass using both the .scss extension. You can use component-level Sass via CSS Modules and the .module.scss extension.

    Before you can use Next.js' built-in Sass support, be sure to install sass. Follow the instructions found in Next.js' documentation.

    Install Normalize.css

    It's important that projects have the same starting point for browsers when rendering elements. Entermedia uses Normalize.css to do this to make elements render more consistently and in line with modern standards.

    Install Storybook

    Entermedia uses Storybook for building UI components and pages in isolation, allowing for streamline UI development, testing, and documentation.

    Follow these instructions to get Storybook up and running in the Next.js project.

    Once Storybook is installed, add the Next.js addon.

    Having a problem running Storybook? There's are a few known issues when running yarn storybook:

    Error: Cannot find module 'webpack/lib/util/makeSerializable.js'

    If you run into this, you'll need to install webpack 5 as a dev dependency:

    npm install webpack@5.68.0 --save-dev

    You may also see this error:

    ModuleNotFoundError: Module not found: Error: Can't resolve '@storybook/addon-docs'...

    If you run into this, you'll need to install the @storybook/addon-docs package as a dev dependency:

    npm install @storybook/addon-docs --save-dev

    Update Storybook Config & Remove Example Stories

    Once Storybook has been installed, update the .storybook/main.js config to look for stories in the components directory:

    stories: [
    "../stories/**/*.stories.mdx",
    "../stories/**/*.stories.@(js|jsx|ts|tsx)",
    "../components/**/*.stories.mdx",
    "../components/**/*.stories.@(js|jsx|ts|tsx)",
  ],

    Be sure to include the gloabl styles in Storybook's preview.js config:

    // Import global styles
import "normalize.css";
import "../styles/theme/default.scss";
import "../styles/base.scss";

    Install Leasoft

    Entermedia uses Leasot to intelligently parse and output TODOs and FIXMEs from comments in your files. Once installed, add a new npm command in the package.json file:

    "todo": "leasot 'components/**/*.tsx' 'pages/**/*.tsx' 'styles/**/*.scss' 'lib/**/*/ts'"

    Project Structure

    Next.js for Drupal's default project setup will need to be initially cleaned up to follow Entermedia's perferred project structure and to remove unused dependencies.

    • /styles - See the styles structure for Next.js projects

    File Updates

    Once the project has been updated to follow the structure above and needed dependencies installed, some of the starter files will need to be updated.

    • /pages/_app.tsx - Import the normalize.css and default/global stylesheet that all pages will use. 
      // Import global styles
import "normalize.css";
import "../styles/theme/default.scss";
